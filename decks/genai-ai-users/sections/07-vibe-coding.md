### C'est quoi le « vibe coding » ?

<br>

<div class="grid grid-cols-2 gap-8 mt-4 text-sm">

<div>

#### La définition

Décrire ce que vous voulez **en langage naturel**, et laisser l'IA **écrire et exécuter le code** à votre place.

- Vous décrivez l'**intention**, pas le code
- L'IA produit un **résultat exécutable**
- Vous **testez**, vous **ajustez** en dialoguant

<div class="text-xs opacity-70 mt-3">« Je code à l'oreille » : vous guidez la <em>vibe</em>, l'IA fait la technique.</div>

</div>

<div>

#### Ce que vous pouvez produire

- 🧮 Un petit **calculateur** / simulateur
- 📊 Un **graphique** depuis vos données
- 🔄 Une **automatisation** répétitive
- 🌐 Une **mini-page** ou landing de test
- 🧹 Un **nettoyage de données** ponctuel

<div class="text-xs opacity-70 mt-3">Parfait pour le prototype, le jetable, le « juste pour moi ».</div>

</div>

</div>

<div class="grid grid-cols-3 gap-4 mt-6 text-sm">

<div class="border-l-4 border-[#10a37f] pl-3">

</div>

### Puissant, mais avec des garde-fous

<br>

<div class="grid grid-cols-2 gap-8 mt-4 text-sm">

<div class="border-l-4 border-[#10b981] pl-4">

#### ✅ Bon terrain de jeu

- Prototypes & maquettes jetables
- Automatisations **personnelles**
- Analyses **ponctuelles** de données
- Apprendre & expérimenter

</div>

<div class="border-l-4 border-[#e63946] pl-4">

#### ⚠️ Prudence requise

- Ne pas mettre en **production** sans relecture d'un pro
- **Vérifier les résultats** (surtout chiffres & calculs)
- Attention aux **données sensibles** manipulées
- Un code qui « marche » n'est pas un code **sûr**

</div>

</div>

<div class="text-center text-sm mt-6 opacity-70">
Règle : vibe coder pour <strong>vous aider</strong>, pas pour livrer un outil critique à l'entreprise sans validation.
</div>
